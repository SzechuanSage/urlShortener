URL Shortener Microservice

User Stories

  • I can pass a URL as a parameter and I will receive a shortened URL in the JSON response.
  • If I pass an invalid URL that doesn't follow the valid http://www.example.com format, the JSON response will contain an error instead.
  • When I visit that shortened URL, it will redirect me to my original link.

To see all stored URL:

https://fcc-projects-szechuan-sage.c9users.io/all

Sample Output

{"original_url":"https://www.freecodecamp.com","short_url":"fcc-projects-szechuan-sage.c9users.io/716981939"}


To add a new URL:

https://fcc-projects-szechuan-sage.c9users.io/new/https://github.com

Sample Output

{"original_url":"https://github.com","short_url":"fcc-projects-szechuan-sage.c9users.io/-1534095099"}


Invalid original URL are not allowed:

https://fcc-projects-szechuan-sage.c9users.io/new/foo://www.bar.com

Sample Output

{"error":"Wrong url format, make sure you have a valid protocol and real site."}


Invalid short URL are not allowed:

https://fcc-projects-szechuan-sage.c9users.io/12345abc

Sample Output

{"error":"This url is not in the database."}
